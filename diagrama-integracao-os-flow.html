O Módulo OS é a ferramenta exclusiva do gestor de equipes e das equipes para gestão de execução. Dentro dele a OS aparece no Pipeline Kanban, na Agenda, no Gantt e na Lista. Quando finalizada, retorna ao Flow.

O submódulo Faturamento (Billing) dentro do Módulo OS = Fase 9 do Flow de Atendimento. O financeiro usa o Billing para: lançar faturas, agendar datas de pagamento e confirmar recebimentos.

🔄 Ciclo Interno do Módulo OS
ENTRADA (todas recebem MGR-YYYY-MM-NNN):
  📁 OS do Projeto (Fase 6 do Flow) → MGR-2026-06-001 → entra em TRIAGEM
  ⚡ OS Rápida (sem projeto) → MGR-2026-06-002 → entra em TRIAGEM com badge NUMERO_GERADO
  📝 OS Regular (sem projeto) → MGR-2026-06-003 → entra em TRIAGEM
  🔄 OS Reagendada → MGR-2026-06-004 → entra em TRIAGEM

FLUXO NORMAL:
  TRIAGEM → PRE_ORCAMENTO → VISITA → ORCAMENTO → APROVACAO → AGENDADO → EM_EXECUCAO
  Técnico faz check-in → executa tarefas → finaliza questionário

SE CONCLUÍDA COM SUCESSO:
  EM_EXECUCAO → AGUARDANDO_FATURAMENTO → AGUARDANDO_PAGAMENTO → CONCLUIDO
  → TODAS as OS (com projeto ou avulsa) seguem para: Relatório → Faturamento → Concluído

SE BLOQUEADA / PROBLEMAS:
  EM_EXECUCAO → REVISAO (nova coluna)
    • PENDENTE_ADMIN — falta peça/ferramenta → gestor resolve → volta para EM_EXECUCAO
    • EM_REVISAO_TECNICA — problema diferente → gestor reavalia → nova OS ou cancela
    • REAGENDAR — sem acesso/tempo → gestor reagenda → nova OS criada

CANCELAMENTO:
  Qualquer estado → CONCLUIDO com statusOS = CANCELADA
8 regras de integração que garantem que o Módulo OS e o Flow de Atendimento funcionem como um sistema único e sincronizado.
Regra 1
OS do Projeto → Pipeline
Quando um projeto chega na Fase 6 (os_distribuidas) e as OS são confirmadas no ProjectOSDistribuicao, cada OS criada aparece automaticamente na coluna TRIAGEM do Pipeline Kanban.
task.projectId = project.id
Regra 2
OS Avulsa → Mesmo Fluxo Completo
OS criada sem projeto (OS Rápida ou Nova OS) entra no Pipeline normalmente. Ao ser concluída, segue o mesmo fluxo: relatório final ao cliente → faturamento → concluído. Não pula nenhuma etapa.
todas as OS → relatório → faturamento → concluído
Regra 2b
Numeração Sequencial OBRIGATÓRIA
TODA OS criada no sistema recebe número sequencial via gerarNumeroOS(): formato MGR-YYYY-MM-NNN. Garante rastreabilidade. Aplica-se a: OS Rápida, OS Regular, OS de Projeto, OS Reagendada.
gerarNumeroOS() → MGR-2026-06-001
Regra 3
OS Concluída → Avança Projeto
Quando todas as OS de um projeto chegam em CONCLUIDO no Pipeline, o sistema avança automaticamente o projeto de os_distribuidas → em_execucao → relatorio_enviado (Fases 7→8).
if (allOsConcluidas) advancePhase(project)
Regra 4
Progresso OS → Progresso Projeto
O percentual de conclusão das OS (X de Y concluídas) atualiza automaticamente os campos totalOSConcluidas e a barra de progresso do projeto na Fase 6-7 do Flow.
project.totalOSConcluidas = osConcluidasCount
Regra 5
Coluna REVISÃO — Gestor Decide
OS bloqueadas (falta peça, problema técnico, sem acesso) vão para coluna REVISÃO. O gestor decide: reenviar para execução, reagendar, cancelar, ou resolver e faturar. O projeto NÃO avança enquanto houver OS em REVISÃO.
workflowStatus = REVISAO → projeto trava
Regra 6
statusOS como Sub-Estado
workflowStatus = posição no Kanban (coluna).
statusOS = detalhe/motivo dentro daquela coluna.
Cada coluna tem statusOS válidos definidos. Combinações inválidas são rejeitadas.
VALID_STATUS_OS_MAP[workflowStatus]
Regra 7
Todas statusOS em UPPERCASE
Unificar formato: apenas valores UPPERCASE (CONCLUIDA, REAGENDAR, etc). Valores legados lowercase são normalizados automaticamente na leitura via normalizeStatusOS().
normalizeStatusOS('concluida') → 'CONCLUIDA'
Regra 8
statusHistory — Auditoria Completa
Toda mudança de workflowStatus é registrada com timestamp + quem mudou. Transições automáticas marcadas como 'auto'. Visível no futuro em timeline da OS.
statusHistory: arrayUnion({status, changedAt, changedBy})
Regra 9
Billing = Fase 9 (Faturamento)
O submódulo Billing do Módulo OS é a ferramenta da Fase 9 do Flow de Atendimento. TODAS as OS (projeto e avulsa) chegam no Billing para o financeiro: lançar fatura, agendar datas de pagamento programadas e confirmar recebimentos. Quando todas as OS de um projeto estão pagas, o projeto avança para concluído.
Billing.tsx = Fase 9 | Tab "Para Faturar" + "Ag. Pagamento" + "Calendário"
Regra 10
Billing → Sync com Projeto
Quando o financeiro confirma pagamento de uma OS vinculada a projeto, o sistema atualiza project.valorRecebido. Quando 100% das OS do projeto estão pagas, o projeto avança automaticamente de em_faturamento → concluido.
allOSPagas → advancePhase(project, 'concluido')
