🔍 10 pontas soltas identificadas por análise estática do codebase. Ordenadas por prioridade de negócio. Itens marcados como Alta impactam diretamente a operação e devem ser resolvidos nos próximos sprints.
ID Prioridade Descrição do Gap Módulos Afetados Sprint Sugerida
GAP-1 Alta Upload de evidências na OS Rápida (F2)
O schema já tem o campo evidencias[] na Task, mas não existe UI para upload de fotos/arquivos na OS Execução. Técnicos não conseguem registrar prova do serviço realizado. 		Pipeline OS Execução Firebase Storage F2
GAP-2 Alta OSEditModal — tela de preenchimento completo da OS Rápida (F3)
OS criadas via OS Rápida têm dadosCompletos: false. Não existe modal/tela para preencher cliente, serviço, valor e demais dados obrigatórios. O botão "Editar" no card da Pipeline está desabilitado até isso ser implementado. 		Pipeline OSEditModal (não existe) Billing F3
GAP-3 Alta ProjectHub isolado — osIds não populados ao criar OS
ProjectHub.tsx não referencia a collection tasks. Quando uma OS é criada para um projeto, o array osIds no documento do projeto não é atualizado automaticamente. Projetos e OS vivem em silos. 		ProjectHub Pipeline criarOSRapida() F4
GAP-4 Alta Estoque isolado — sem dedução automática quando materiais são usados em OS
Inventory.tsx gerencia peças mas não tem nenhuma ligação com OS Execução. Materiais consumidos em campo não debitam o estoque, gerando inconsistência entre físico e sistema. 		Inventory OS Execução tasks.materiais[] F4 / F5
GAP-5 Média Ponto/RH — horas da OS não alimentam banco de horas
O campo horasExecutadas existe na Task mas não é lido pelo módulo de PointSystem. Horas trabalhadas em campo não contam para o banco de horas do técnico. 		PointSystem OS Execução horasExecutadas F5
GAP-6 Média Billing → FluxoCaixaGerencial — caminho de aggregação não validado end-to-end
A conexão existe no código mas não foi testada com dados reais de ponta a ponta. OS concluída → conta a receber no Billing → aparece no FluxoCaixa não está confirmado funcionando. 		Billing FluxoCaixaGerencial Firestore queries Hotfix / teste
GAP-7 Média Intelligence/BI — análises não geram OS automaticamente
O módulo de inteligência identifica oportunidades (ex: cliente sem visita há 6 meses) mas não tem um botão/ação para criar uma OS diretamente a partir da análise. Fluxo é manual. 		Intelligence criarOSRapida() Pipeline F5 / F6
GAP-8 Média Histórico de OS no Cliente não é auto-atualizado
O documento do cliente tem campo historicoOS mas ao criar ou concluir uma OS, esse array não é atualizado automaticamente. O histórico fica desatualizado. 		Clients criarOSRapida() criarOS() F3 / F4
GAP-9 Baixa statusHistory gravado mas não exibido em nenhuma tela
O campo statusHistory[] é gravado nas Tasks mas nenhuma tela mostra esse histórico ao usuário. Auditoria de mudanças de estado da OS não está acessível. 		Pipeline OS Execução tasks.statusHistory[] F5 / F6
GAP-10 Baixa minEstoque no Inventory sem alerta visual
O campo minEstoque existe nos itens do inventário mas não há badge/alerta visual quando um item está abaixo do mínimo. Gestão de reposição é totalmente manual. 		Inventory minEstoque Dashboard F5
